## Критические проблемы (3)

### 0. Неполный английский перевод.
**Критичность:** Критическая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Главная не переводится полностью на английский

### 1. Использование React хуков без импорта React
**Файл:** `src/i18n/client.ts` (строки 110-125)
**Критичность:** Критическая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Код использует React хуки (`React.useState`, `React.useEffect`), но не импортирует React. Это вызовет ошибки во время выполнения.
```typescript
// Проблемный код:
export function useTranslation() {
  const [, forceUpdate] = React.useState({}); // React не импортирован

  React.useEffect(() => { // Это упадёт с ошибкой
    const unsubscribe = i18nClient.subscribe(() => {
      forceUpdate({});
    });
    return unsubscribe;
  }, []);
  // ...
}
```
**Влияние:** Полный отказ системы переводов в компонентах, использующих `useTranslation`
**Исправление:** Добавить импорт React в начало файла:
```typescript
import React from 'react';
```

### 2. Несогласованные ключи перевода между файлами
**Файлы:** `src/i18n/ui.ts` и `src/components/DynamicLanguagePicker.astro`
**Критичность:** Критическая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Ключи перевода в DynamicLanguagePicker не соответствуют централизованной системе переводов. Компонент имеет жёстко заданные переводы, отличающиеся от ui.ts.
```typescript
// DynamicLanguagePicker.astro (строка 73):
'nav.about': 'O kompanii',  // Транслитерация, не правильный русский

// ui.ts (строка 10):
'nav.about': 'О компании',  // Правильный перевод
```
**Влияние:** Несогласованные переводы на сайте, кошмар для поддержки
**Исправление:** Удалить жёстко заданные переводы из DynamicLanguagePicker и использовать централизованную систему

### 3. Отсутствие атрибута lang в HTML элементах
**Файлы:** `src/components/DynamicLanguagePicker.astro` (строка 102)
**Критичность:** Критическая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Динамическое изменение языка документа не сопровождается правильной установкой атрибута lang на корневом элементе HTML.
```javascript
// Проблемный код:
document.documentElement.lang = lang; // Хорошо, но нет начальной установки
```
**Влияние:** Нарушение WCAG 3.1.1, плохая поддержка screen readers, некорректная работа языковых функций браузера
**Исправление:** Убедиться, что атрибут lang установлен в Layout.astro и корректно обновляется при смене языка

## Проблемы высокой важности (5)

### 4. Утечка памяти в event listeners
**Файл:** `src/components/DynamicLanguagePicker.astro` (строки 298-332)
**Критичность:** Высокая
**Статус:** ⚠️ ЧАСТИЧНО РЕШЕНО (30.04.2026)
**Описание:** Event listeners добавляются, но никогда не очищаются должным образом при уничтожении компонентов.
```typescript
// Проблема: Добавление listeners без очистки
newBtn.addEventListener('click', handleMenuToggle);
newBtn.addEventListener('touchend', handleMenuToggle);
// Нет механизма очистки
```
**Влияние:** Утечки памяти в SPA приложениях, возможная деградация производительности
**Исправление:** Локальные listeners очищаются через cloneNode, но глобальные click/touchend handlers остаются без очистки

### ~~5. Небезопасное приведение типа~~ ✅ РЕШЕНО
**Файл:** `src/i18n/client.ts` (строка 14)
**Критичность:** Высокая
**Статус:** ✅ РЕШЕНО (30.04.2026)
**Описание:** Небезопасное приведение типа без валидации может вызвать ошибки во время выполнения.
```typescript
const storedLang = localStorage.getItem('user-lang') as Language;
// Нет проверки, что storedLang действительно является Language
```
**Влияние:** Ошибки во время выполнения, если localStorage содержит некорректные данные
**Исправление:** Добавлена валидация: `if (storedLang && storedLang in ui)`

### 6. Дублирование логики переводов
**Файлы:** Несколько файлов
**Критичность:** Высокая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Логика переводов дублируется в:
- `src/i18n/client.ts` (updatePageTranslations)
- `src/components/DynamicLanguagePicker.astro` (updatePageTranslations)
- `src/layouts/Layout.astro` (initializeDynamicTranslations)

**Влияние:** Дублирование кода, накладные расходы на поддержку, потенциальные несоответствия
**Исправление:** Централизовать логику обновления переводов в одном утилитном модуле

### 7. Отсутствие обработки ошибок в вызове GitHub API
**Файл:** `src/pages/index.astro` (строка 15)
**Критичность:** Высокая
**Статус:** ⚠️ ЧАСТИЧНО РЕШЕНО (30.04.2026)
**Описание:** Вызов GitHub API не имеет должной обработки ошибок и механизмов отката.
```typescript
const release = await fetchLatestRelease('bit-Tecnologies/bit_hub', lang).catch(() => null);
// Тихий сбой может быть неподходящим для пользовательского опыта
```
**Влияние:** Плохой пользовательский опыт при недоступности GitHub API
**Исправление:** Есть базовый .catch с fallback на null, но нет визуальной обратной связи пользователю

### 8. Отсутствие оптимизации изображений и WebP формата
**Файлы:** `astro.config.mjs` (строки 79-81), `public/` директория
**Критичность:** Высокая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Astro image optimization настроена, но отсутствует конвертация в современные форматы (WebP, AVIF) и оптимизация размеров.
```javascript
// astro.config.mjs:
image: {
  domains: ['cybernattor.github.io'] // Только домены, нет оптимизации
}
```
**Влияние:** Больший размер бандла, медленная загрузка изображений, плохой Core Web Vitals (LCP)
**Исправление:** Включить автоматическую конвертацию в WebP/AVIF, добавить responsive изображения с правильными размерами

### 13. Отсутствие skip link для keyboard навигации
**Файлы:** `src/layouts/Layout.astro`
**Критичность:** Средняя
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Нет skip link для быстрого перехода к основному контенту для пользователей клавиатуры.
```html

```
**Влияние:** Нарушение WCAG 2.4.1, плохой UX для пользователей клавиатуры и screen readers
**Исправление:** Добавить skip link в начале Layout.astro с правильными стилями

### 14. Недостаточная контрастность текста в dark/light режимах
**Файлы:** Tailwind CSS конфигурация и компоненты
**Критичность:** Средняя
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Некоторые текстовые элементы могут не соответствовать WCAG AA требованиям контрастности (4.5:1 для нормального текста).
```css
/* Потенциальные проблемы с контрастностью */
text-slate-300 на dark:bg-slate-800
text-slate-700 на light:bg-white
```
**Влияние:** Нарушение WCAG 1.4.3, плохая читаемость для пользователей с нарушениями зрения
**Исправление:** Провести аудит контрастности и использовать более контрастные цвета

**Файлы:** CSS и Tailwind конфигурация
**Критичность:** Низкая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Нет медиа-запроса для prefers-reduced-motion, что может вызывать проблемы у пользователей с вестибулярными расстройствами.
```css
/* Отсутствует */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
**Влияние:** Нарушение WCAG 2.3.3, плохой UX для пользователей с чувствительностью к движению
**Исправление:** Добавить Tailwind плагин или кастомные стили для prefers-reduced-motion

### 20. Отсутствие preload для критических ресурсов
**Файлы:** `src/layouts/Layout.astro`, `astro.config.mjs`
**Критичность:** Низкая
**Статус:** ✅ ПОДТВЕРЖДЕНО
**Описание:** Критические CSS и шрифты не имеют preload тегов, что замедляет First Contentful Paint.
```html
<!-- Отсутствуют preload -->
<link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/styles/main.css" as="style">
```
**Влияние:** Медленная загрузка, плохой Core Web Vitals (FCP, LCP)
**Исправление:** Добавить preload для шрифтов и критического CSS в Layout.astro
